    msxm13.dll错误“80070005”

    发表于 2025-12-22

    asp 启用Https以后后台出现功能问题，报错msxm13.dll错误“80070005”

    问题原因

    核心原因是组件版本不兼容 HTTPS、协议不匹配、证书信任问题（不是传统文件权限）
    Msxml2.XMLHTTP是客户端组件，服务器端用它请求 HTTPS 链接会有兼容性 / 权限问题；服务器端应使用Msxml2.ServerXMLHTTP（高版本），它才支持 HTTPS 协议

    解决步骤

    1. 替换 HTTP 请求组件（用服务器端高版本）
      把Msxml2.XMLHTTP换成Msxml2.ServerXMLHTTP.6.0（需确保服务器安装了 MSXML 6.0
    2. 修正 URL 协议为 HTTPS
      将代码里的http://改成https://，匹配网站部署协议
